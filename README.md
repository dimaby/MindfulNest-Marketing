# MindfulNest — маркетинговый микросайт

Одностраничный сайт для исследования рынка идеи **MindfulNest** — структурированного рабочего пространства для покупателя недвижимости и его риэлтора (альтернатива WhatsApp-хаосу при параллельном просмотре десятка объектов).

Сайт одновременно выполняет три задачи:

1. **Лендинг** — презентует продукт и проблему для практикующих агентов.
2. **Отчёт** — проводит по 8 ключевым экранам мобильного приложения (сторона покупателя) с объяснением фич.
3. **Опрос** — 14 вопросов (тест PMF по Шону Эллису + валидация проблемы/решения + готовность платить), который заполняют действующие агенты.

Всё — на русском. Хостинг — **GitHub Pages**, без бэкенда.

---

## Стек

- Чистый статический HTML/CSS/JS в одном файле `index.html`.
- Без сборщиков, без фреймворков, без `package.json`.
- Шрифты подгружаются из Google Fonts (`Space Grotesk` для заголовков, `Inter` для текста).
- Скриншоты приложения лежат в `assets/` (`01-login.png` … `08-my-agents.png`).
- Отправка опроса → **Formspree** (эндпоинт `https://formspree.io/f/mvzdwjpn`).

Визуальный стиль вдохновлён референсом `https://kishkisalexandra.github.io/ArchiveCSP/`: тёмно-синий фон, акценты фиолетовый (`#7C5CFF`) + бирюза (`#4DD0C1`), скруглённые карточки с мягкими градиентами и свечением.

---

## Локальный запуск

Любой статический сервер подойдёт. Из корня репозитория:

```bash
python3 -m http.server 8080
# дальше открой http://localhost:8080
```

Шага сборки нет — правишь `index.html`, обновляешь вкладку в браузере.

---

## Деплой на GitHub Pages

1. Запушить репозиторий на GitHub.
2. В репозитории: **Settings → Pages → Source**: ветка `main`, папка `/ (root)`.
3. Сайт будет доступен по адресу `https://<user>.github.io/<repo>/` — `index.html` уже точка входа.

## Структура страницы

`index.html` читается сверху вниз как один скролл:

- **`<nav class="top">`** — липкий полупрозрачный хедер с якорными ссылками.
- **`<header class="hero">`** — eyebrow-плашка, градиентный заголовок, тизер проблемы, мета-статы, таб-навигация, 4 статистические карточки.
- **`#problem`** — сплит-сравнение (WhatsApp сегодня vs MindfulNest).
- **`#solution`** — 5 фич-строк, каждая пара «скриншот экрана + объяснение».
- **`#screens`** — галерея из 8 мокапов, каждая плитка открывает полноразмерный PNG.
- **`#insights`** — сетка из 6 карточек с WOW-отличиями.
- **`#api`** — секция про API-коннектор и интеграции (AmoCRM, Bitrix24, порталы, WhatsApp, Google, Zapier).
- **`#survey`** — сам опрос из 4 блоков / 14 вопросов. Шкала 1–10 для Q4 генерируется на JS.
- **`<footer>`**.

Весь CSS — один `<style>`-блок в начале файла. Палитра через CSS-переменные в `:root` (`--violet`, `--teal`, `--bg`, …) — меняешь их, и тема меняется везде сразу.

JS в конце файла делает: клик по табу → плавный скролл; `IntersectionObserver` синхронизирует активный таб со скроллом; строит шкалу 1–10; перехватывает submit, пишет резервную копию в `localStorage` и отправляет данные в Formspree.

## Опрос и Formspree

Сабмит:

1. Сохраняет резервную копию в `localStorage` под ключом `mindfulnest_responses`.
2. Собирает читаемый payload через `buildFormspreePayload()`:
   - Сырые слаги (`yes-constant`, `somewhat`, …) переводятся в полные русские фразы через объект `MAP`.
   - Ключами служат полные тексты вопросов с префиксом-номером (`"01. Узнаёте ли Вы проблему..."`, `"06a. 3 самые полезные функции"`, `"06b. Другое (функция)"` и т. д.) — чтобы в дашборде Formspree и в письме всё было осмысленно.
3. POST JSON'ом с заголовками `Content-Type: application/json` + `Accept: application/json`, чтобы Formspree отвечал JSON'ом, а не 302-редиректом.
4. Служебные поля Formspree: `_subject` (тема письма) и `_replyto` (ставится только если контакт — валидный email).
5. На успех: форма скрывается, показывается `#thanks`. На ошибку: кнопка разблокируется, показывается `.submit-error` с fallback-mailto на `me@dimaby.com`.

**Обязательные поля.** У radio-групп Q1, Q2, Q4, Q5, Q9, Q11, Q14 стоит `required` — без них форма не отправляется. Остальное (чекбоксы, textarea, контакты) оставлено необязательным, чтобы не резать конверсию.

**Переименование поля.** Если меняешь `value` у какой-то опции в HTML, не забудь обновить тот же ключ в `MAP`, иначе в дашборде появится сырой слаг вместо фразы.

**Лимиты.** Бесплатный Formspree — 50 сабмитов в месяц. Если упрёмся в потолок: платный тариф, Cloudflare Worker → Airtable, либо Google Apps Script.

## Правила при редактировании

- **Не разносить на несколько файлов**, если пользователь явно не попросил. Один файл — сознательное решение: моментальный деплой на GitHub Pages и быстрая загрузка.
- **Не тащить сборщики** (webpack/vite/npm) — только голый HTML/CSS/JS.
- Если добавляешь/удаляешь вопрос в опросе, обновляй и разметку в `#survey`, и логику в `collectAnswers()` / `buildFormspreePayload()` / `MAP`, если поменялись имена полей.
- Тёмная тема с фиолетовым и бирюзой — намеренная. Не «модернизировать» её в светлую без подтверждения.
- Скриншоты в `assets/` — источник правды по виду приложения. Если появляется новый экран, клади его в `assets/` с пронумерованным именем и добавляй и в соответствующую фич-строку (если релевантно), и в галерею `#screens`.
